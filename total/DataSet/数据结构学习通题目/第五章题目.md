### 1.![alt text](<PicturE/屏幕截图 2026-01-03 162214.png>)

在图论中，对于一棵有 \(n\) 个结点的无向树，边数为 \(n-1\)，而所有结点的度数之和等于边数的两倍，因此度数和为 \(2(n-1) = 2n-2\)。

然而，在某些课程（如数据结构）中，若将树视为有根树，并将结点的“度数”定义为子结点个数，则所有结点的度数之和等于边数，即 \(n-1\)。学习通提供的答案 \(n-1\) 可能源于后一种定义。

建议根据具体课程内容判断，但图论标准答案为 \(2n-2\)。

### 3.![alt text](<PicturE/屏幕截图 2026-01-03 162810.png>)![alt text](<PicturE/屏幕截图 2026-01-03 163414.png>)

对于完全二叉树，叶结点数 \(m\) 与总结点数 \(n\) 满足关系：\(m = \lceil n/2 \rceil\)。

给定 \(m = 124\)，则：
- 若 \(n = 247\)，\(m = \lceil 247/2 \rceil = 124\)；
- 若 \(n = 248\)，\(m = \lceil 248/2 \rceil = 124\)；
- 若 \(n = 249\)，\(m = \lceil 249/2 \rceil = 125\)；
- 若 \(n = 250\)，\(m = \lceil 250/2 \rceil = 125\)。

因此，当叶结点数为 124 时，总结点数最大为 248。

无论求最多还是最少，都是向上取整。

### 4.![alt text](<PicturE/屏幕截图 2026-01-03 163604.png>)

#### 4.1解答

对于高度为 \(h\)（根结点为第1层）的完全二叉树，若第6层有24个叶结点，则树的最大可能层数为7层（因为叶结点只能出现在最下两层，第6层为倒数第二层）。此时，前5层为满二叉树，结点数为 \(2^5 - 1 = 31\)；第6层最多有 \(2^{5} = 32\) 个结点，其中24个为叶结点，则非叶结点为 \(32 - 24 = 8\) 个，每个非叶结点产生两个第7层结点，故第7层有 \(8 \times 2 = 16\) 个结点。总结点数最大为 \(31 + 32 + 16 = 79\)。

**答案：B**

---

#### 4.2解答

对于有1000个结点的完全二叉树，计算其高度 \(h\) 满足 \(2^{h-1} \le 1000 < 2^h\)，得 \(h = 10\)。前9层满二叉树结点数为 \(2^9 - 1 = 511\)，第10层结点数为 \(1000 - 511 = 489\)。第9层（次底层）满时有 \(2^{8} = 256\) 个结点，其中产生第10层结点的父结点数为 \(\lceil 489 / 2 \rceil = 245\)，故第9层叶结点数为 \(256 - 245 = 11\)。

**答案：A**

#### **5.1**  
题意：  
- 完全二叉树结点数为 \(n\)。  
- 存储在数组 \(R[0 \dots n-1]\) 中。  
- 根结点为 \(R[0]\)。  
- 已知 **结点 \(R[i]\)** 有左孩子，问左孩子的下标。  

标准完全二叉树顺序存储的规则是：
对结点 \(R[k]\)：
- 左孩子下标为 \(2k+1\)（如果 \(2k+1 < n\)）。
- 右孩子下标为 \(2k+2\)（如果 \(2k+2 < n\)）。  

注意题目给的例子是“若结点 \(R[i]\) 有左孩子”，并且答案选项是关于 \(i\) 的表达式。  
选项里：
A. \(R[2i-1]\)  
B. \(R[2i]\)  
C. \(R[2i+1]\)  
D. \(R[2i+2]\)  

显然，根索引从 0 开始，左孩子公式为 \(2i+1\)。  
所以正确答案是 **C**。

---

#### **5.2**  
“若结点 \(R[i]\) 有右孩子，则右孩子是”——右孩子下标为 \(2i+2\)。  
所以正确答案是 **D**。

### 6.![alt text](<PicturE/屏幕截图 2026-01-03 164648.png>)

在二叉树的前序、中序、后序遍历中，**叶结点的相对顺序**是否改变取决于遍历方式。

---

考虑一个简单例子：  
一个根结点，左右孩子都是叶结点。

- 树结构：
```
   1
  / \
 2   3
```
叶子是 2 和 3。

**前序**：1 2 3（叶子顺序 2 → 3）  
**中序**：2 1 3（叶子顺序 2 → 3）  
**后序**：2 3 1（叶子顺序 2 → 3）  

这里顺序没变。

---

再考虑一个复杂的例子：  
树结构：
```
   1
  / \
 2   3
    / \
   4   5
```
叶子是 2、4、5。

**前序**：1 2 3 4 5（叶子顺序 2, 4, 5）  
**中序**：2 1 4 3 5（叶子顺序 2, 4, 5）  
**后序**：2 4 5 3 1（叶子顺序 2, 4, 5）  

叶子相对顺序依然没变。

---

可以证明：  
因为前序、中序、后序都是深度优先遍历（DFS），三种遍历中，所有叶结点的**出现顺序**都按“从左到右”的次序进行。具体来说：  
- 叶子 A 在叶子 B 的左边（按树的结构），那么在任何一种 DFS 遍历中（无论是前序、中序、后序，只要先左后右），A 都会比 B 先被访问到。  
因此叶结点的**相对顺序**在三种遍历中**始终相同**。

---

答案选 **B. 不发生变化**。
